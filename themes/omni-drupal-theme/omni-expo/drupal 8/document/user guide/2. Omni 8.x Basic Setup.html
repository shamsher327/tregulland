Omni Basic Setup

- Introduction

In this section, we will show you some basic setup steps that Omni theme need. By using this guide, we assume you have basic knowledge of Drupal and have Drupal 8.x installed on your system.

- Theme Pack Folder Structure

  • document
    • user guide
    • views: importable view files
    • sample content: demo SQL files & demo image files
  • licensing
  • modules: all required modules for Omni Drupal theme.
  • theme: theme files folder
    • omni_expo: this is the actual theme, please upload this folder to into drupal 8 root's themes/custom folder.
  • README.txt

- Installation

There is a modules folder within the download package which include all necessary modules for this theme. Only package included version of modules were tested for compatibility. If new version released please backup before upgrade since it might not be compatible.

Please install Omni Expo theme and following Modules (If you don't know how to install Drupal theme, please check our Drupal Quick Start guide):

  • Admin Toolbar 8.x-1.17
  • Configuration Update Manager 8.x-1.1
  • Contact Block 8.x-1.2
  • Features 8.x-3.0-rc1
  • Field Group 8.x-1.0-rc4
  • Inline Entity Form 8.x-1.0-alpha6
  • Previous/Next API 8.x-2.x-dev
  • Share Buttons by AddToAny 8.x-1.14
  • Social Media Links Block 8.x-2.1
  • Simplenews 8.x-1.0-alpha2
  • Views Infinite Scroll 8.x-1.2
  • Omni Custom
  • Omni Feature: simply import all content type & block type for Omni theme. Available within download files.

- Basic Configuration

You need to do ALL the following steps by order, even if you think you might not need it for now.

1. Enable Omni Expo theme

Go to Appearance

  • Enable and set Omni Expo as Default theme
  • Make sure administration theme set to Seven

2. Enable required modules by order!

Go to Extend page:

  1. Enable modules Statistics, Update Manager, Admin Toolbar, Contact Block, Field Group, Inline Entity Form, Simplenews, AddToAny, Previous/Next, Social Media Links Block, Views Infinite Scroll then click Save Configuration
  2. Enable modules Features, Features UI, Configuration Update Base and Omni Custom then click Save Configuration
  3. Enable Omni Feature then click Save Configuration (might take longer).

3. Setup avatar & author block

Go to Configuration > Account setting > Manage fields

  • Edit Picture field
  • Upload images/avatar.png file as default image
  • Click Save Configuration

Go to Configuration > Account setting > Manage form display

  • Enable Title & Profile for default display then Save.
  • Edit Admin profile and input some sample data for Title & Profile fields then Save.

Go to Structure > Block layout

  • At Author region, click Place block button and select View Block: About author.
  • Make sure it only display on Post content type.

4. Change front page path

Go to Configuration > Basic site settings > Front Page

  • Change default front page field from /node to /post
  • Click Save Configuration

5. Adding taxonomy terms for Category

Go to Structure > Taxonomy > Category

  • Click list terms button and adding some sample terms. For example: city, life, style etc.

6. Views

Go to Structure > Views

  • Disable Custom block library
  • Edit Taxonomy term view and remove Header (Global: Rendered entity - Taxonomy term (Global: Rendered entity - Taxonomy term)) then save it.

For live site, you should enable Aggregate for JavaScript and CSS files. Now you have completed the basic setup! You can continue on Omni Feature Setup.
