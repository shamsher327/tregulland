New Content Type

1. Post

The new content type Post is very important for Omni Expo. It has same process to create a Article node with a few extra setting options. All nodes seen on demo created by Post. Go to Add content > Click on Post.

Post require thumbnail image (best size 460X400) uploaded for all teaser images and image field (best size 1920X600) for full node heading.

At specific setting, user can set where this post node should appear. For example, mega menu, trending slider, highlight etc. Multiple choice!

Menu

1. Main navigation

Fairly simple, Go to Structure > Menus > Main navigation > list links and add new links for Main navigation. Go to Structure > Blocks find Main navigation block. Publish it at Menu main block region.

2. Main navigation mobile

Omni Expo require menu set for mobile version so when mobile device viewing the website, Main navigation mobile menu will show instead of Main navigation menu.

After basic setup steps, your site will have Main navigation mobile menu in place already. You only need to modify it to fit your needs.

3. Mega menu

To creat Mega menu:

  • Go to Structure > Block Layout > Custom block library > Add custom block
  • Select Mega menu
  • Click add new custom block button
  • Fill in Block description, Link URL & text then attach a menu item
  • Menu item field can ONLY pick up the block has been published/placed. Those blocks can be placed into Usable region. Usable region act like place holder and will not show on any pages.

For example, below screenshot is Menu item: Mega Menu item - Motor. Mega Menu item - Motor include two parts, Mega menu - motor (created by using Structure > Menus) and View Block: Mega menu 1 (created by Omni Expo views). Those two parts have been merged by Custom Block merging function. See Custom Block section below.

Repeat above steps to creat more mega menu links.

Showcase

Omni Expo offer one Showcase style with easy setup steps. By using two new Block types: Showcase and Showcase Element, user can quickly create Showcase just like to create any other blocks. There is also a Showcase block region for placing showcase block.

- Understand Showcase and Showcase Element block type

  • Showcase: contain basic Showcase setting. It also contain multiple Showcase Element.
  • Showcase Element: contain actual image, slide title, link, caption etc.

Each Showcase Element act like slide and Showcase act like container that glue slides together.

1. Main

  • Requirement:
    • Complete General setting.
    • Images size normally 1920px X 1000px as background.
    • Position
    • Category
    • Pattern
    • Link or Button
  • Optional:
    • Filter
  • Suitable regions: Showcase region only

Custom Block

It is very important for user to understand how Omni Expo's Custom Block work. A lot of features depend on Custom Block

In Omni Expo there are two types of block:

  1. Custom Block type
  2. Non-Custom Block type (system block, menu block, basic block etc.)

Omni Expo using Custom Block's merging feature to merge up to six other blocks' content together to get flexible layouts and styles.

Custom Block use Attachment feature turn Non-Custom Block into Custom Block.
To use Merging or Attachment feature, All Block must be published/placed first.

For example, User account menu is a Drupal 8 menus block which is a Non-Custom Block. If user want to merge User account menu block with other blocks. First it need to be published then attached to a Custom Block, last step to merge with others.

- Merging

The idea is to merge several blocks into one block by horizontal. Use it only when you want to merge blocks.

  • Type: MUST select a correct merging type
  • Horizontal Setting
    • even width columns: checked by default or user can setup each column size by using Size option. All merged columns size in total should be 12. For example, four merged columns size can be 3+3+3+3 or 4+4+2+2
    • equal height columns: force all merging blocks into same height. Required each individual block checked with this options.
    • collapse: by default each merging blocks will have space in-between. Checking this box will collapse the gap.

Merging field can ONLY pick up the block has been published/placed. Those blocks can be placed into Usable region. Usable region act like place holder and will not show on any pages.

- Attachment

The idea is to attach any blocks that have been published/placed into Block layout and turn it into Custom block so other Custom block can use it to merge.

Attachment field can ONLY pick up the block has been published/placed. Those blocks can be placed into Usable region. Usable region act like place holder and will not show on any pages.



Why Merging

Layout & style

To use Drupal 8 bartik theme as example, there are Featured bottom first, Featured bottom second, Featured bottom third, three Featured bottom regions. If we fill in all three Featured bottom regions then Featured bottom will always show as three columns layout. Unlike bartik, in Omni Expo there is only one Main bottom region. If user use Main bottom region with merging feature then Omni Expo can have one stack as three columns and put another merging stack below as four columns and so on. Each stack can have its' own background colors, background image, title, styles etc. Result Omni Expo can use much less block regions with more layout options and styles.



Why Attachment

Turn any block into Custom Block, So user can use Custom Block setting

Views

- Omni custom block library

Omni custom block library provide an alternative view to Drupal 8 built-in Custom block library view. User can disable the Drupal original Custom block library view.

Omni custom block library view allow user to select block type and only display the specific block type content.

- Ready to use blocks

Omni Expo has many ready to use blocks. It can be found after click Place Block button at Block Layout. Type view into search box.

Contact Form

- Contact Form Block

Contact Form Block require Contact Block module installed & enabled. Go to Structure > Block Layout and click Place block button then search for Contact block and select Omni Custom contact form as Contact form option.
