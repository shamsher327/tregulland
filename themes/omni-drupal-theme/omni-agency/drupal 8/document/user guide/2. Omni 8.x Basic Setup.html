Omni Basic Setup

- Introduction

In this section, we will show you some basic setup steps that Omni theme need. By using this guide, we assume you have basic knowledge of Drupal and have Drupal 8.x installed on your system.

- Theme Pack Folder Structure

  • theme: theme files folder
    • omni_agency: this is the actual theme, please upload this folder to your server.
  • licensing
  • modules: all required modules for Omni Drupal theme.
  • document
    • user guide
    • views: importable view files
    • sample content: demo SQL files & demo image files
  • README.txt

- Installation

There is a modules folder within the download package which include all necessary modules for this theme. Only package included version of modules were tested for compatibility. If new version released please backup before upgrade since it might not be compatible.

Please install Omni Magazine theme and following Modules (If you don't know how to install Drupal theme, please check our Drupal Quick Start guide):

  • Admin Toolbar 8.x-1.15
  • Configuration Update Manager 8.x-1.1
  • Contact Block 8.x-1.1
  • Features 8.x-3.0-beta3
  • Field Group 8.x-1.0-rc4
  • Inline Entity Form 8.x-1.0-alpha6
  • Share Buttons by AddToAny 8.x-1.10
  • Social Media Links Block 8.x-2.1
  • Omni Custom
  • Omni Feature: simply import all content type & block type for Omni theme. Available within download files.

- Basic Configuration

You need to do ALL the following steps, even if you think you might not need it for now.

1. Enable required modules BY ORDER!

Go to Extend page:

  1. Enable modules Admin Toolbar, Contact Block, Field Group, Inline Entity Form, AddToAny, Social Media Links Block then click Save Configuration
  2. Enable modules Features, Features UI and Configuration Update Base then click Save Configuration
  3. Enable modules Omni Custom and Omni Feature then click Save Configuration

2. Enable Omni Agency theme

Go to Appearance

  • Enable and set Omni Agency as Default theme
  • Make sure administration theme set to Seven
  • Click Setting link under Omni Agency
  • Omni Setting
    • Sticky header: checked
    • Reveal mobile menu: checked
  • Save the change

3. Configure Blocks

Go to Structure > Block layout

  • Move Search form block from Site Info into Search region & disable its' title.
  • Move Footer menu block from Site Info into Closure region.
  • Move User login block into Login region & disable its' title.
  • Disable Tools, Powered by Drupal blocks from Site Info region.
  • Configure Site branding block. Toggle Branding Elements: Check Logo & Slogan OR Site name & Slogan. It can't display Logo & Site Name at same time.
  • Configure Main navigation block. Change Maximum number of menu levels to display as 3, then Save block.
  • Configure Page title block. Click on Pages tab then fill in /node/* and select Hide for listed pages
  • Place Social Media Links block. Click on Place block button beside the Social media region then place Social Media Links block. Disable its' title and fill in your social media links. Appearance as horizotal and Icon Sets as Font Awesome with fa-lg style only then save it.
  • Keep the rest of blocks as it is.

4. Configure AddToAny

Go to Home > Administration > Configuration > System > AddToAny

  • Change Icon size to 16 pixels
  • Adding Service Buttons HTML code
    <a class="a2a_button_facebook"><span class="fa fa-facebook"></span></a>
    <a class="a2a_button_twitter"><span class="fa fa-twitter"></span></a>
    <a class="a2a_button_google_plus"><span class="fa fa-google-plus"></span></a>
    <a class="a2a_button_pinterest"><span class="fa fa-pinterest-p"></span></a>
    <a class="a2a_button_linkedin"><span class="fa fa-linkedin"></span></a>
  • Check Universal Button as None
  • Open Placement and check only Article, Blog, Display for node teasers, Display in content section
  • Click Save Configuration

5. Change default contact form

Go to Home > Administration > Structure > Contact forms > Edit Custom contact form and check Make this the default form

6. Import views

Go to Configuration > Configuration synchronization > Import tab > Single item

  • Select View for Configuration type
  • Open omni-drupal-theme/omni-agency/drupal 8/document/views/views.view.omni_custom_block_library.yml file
  • Copy & Paste views.view.omni_custom_block_library.yml file content into Paste your configuration here box
  • Click Import then Confirm
  • Do the same for views.view.omni_library.yml

Go to Structure > Views. There should be two more views that you just imported:

  1. Omni custom block library
  2. Omni library

Now you can disable Custom block library Drupal built-in view since we have just imported Omni custom block library view which can replace its' function with more options.

7. Setup avatar

Go to Configuration > Account setting > Manage fields

  • Edit Picture field
  • Upload images/avatar.png file as default image
  • Click Save Configuration

For live site, you should enable Aggregate for JavaScript and CSS files. Now you have completed the basic setup! You can continue on Omni Feature Setup.
