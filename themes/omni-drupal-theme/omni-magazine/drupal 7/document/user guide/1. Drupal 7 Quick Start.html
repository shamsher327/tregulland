1. Drupal Quick Start

Introduction

If you are new to Drupal, the amount of information you can get from Drupal.org can be overwhelming. This is a 10 minutes beginner guide. It explain the very basic and give you a quick start on Drupal 7.x. If you are familiar with Drupal, you can skip to Omni Basic Setup user guide.

Installation

1. Drupal core

You can get the latest Drupal core files here. Drupal installation process is very similar to other CMS (Content Management System) like Wordpress and Joomla.

Requirements:

  • Host ready
  • Database created
  • Admin username & password for the database
  • Drupal core file uploaded & uncompressed to the desire directory

Key in your site base URL into browser, follow the steps process and you will have Drupal 7 installed. For more detailed documents on Drupal installation, you can check out here.

2. Drupal modules & themes

Drupal module serve the same purpose as Wordpress plug-in or Joomla extensions. Basically it extend the core functions so you can have features like showcase, tabbed blocks etc.

Before install Drupal modules & themes, you need to complete the Drupal core installation. Drupal 7.x directories structure as below:

  • includes
  • misc
  • modules
  • profiles
  • scripts
  • sites
  • themes
Directory modules & themes is for the system, you shouldn't install your modules & themes there. Instead you need to go into sites directory and create a directory call all if there isn't one. Create a directory call modules, upload all your modules to this directory. Create a directory call themes, upload your theme files to this directory.

Result:

  • ...
  • sites/all/modules/...
  • sites/all/themes/...
  • ...
You can go to Home > Administration > Modules, to enable those modules and go to Home > Administration > Appearance, enable and set default theme. You can also find alternative documentation on this subject at here.

Please don't enable any modules & theme just yet, you need to do it at the Omni Basic Setup

Drupal Basic

1. What is what?

  • Dashboard - Gathering overall information. Can be customized.
  • Content - List all the nodes ( node is a Drupal term for post or article ) you have created.
  • Structure - Site Structure. Including important units of Blocks, Content types, Menus, Taxonomy & Views.
  • Appearance - Themes & theme settings.
  • People - Users & permissions.
  • Modules - Core function extension union.
  • Configuration - Site Configuration. Including important units of Text formats, Image styles, Site information, Backup and Migrate, Cron, Performance etc.
  • Report - Site log, status, updates info.
  • Add content - Create a new node.

2. Find your way

  • Create a new post: Click on Add content. You will have options of Article, Basic page, Blog entry (Blog module need to be enabled) & Showcase (custom content type for Omni theme) etc. For a regular post select Article content type.
  • Create a new menu item: Click on Structure then Menus. Main menu is primary menu. Click on list links for existing menu items. Click on add link to add a new menu items. If a menu item has submenu items, that menu item need to be edited and enabled the Show as expanded option.
  • Create a new block: Click on Structure then Blocks. Click on Add Block link to add a new block.

3. Have to know - Block

You have to know how block works because from content aspect, everything build by blocks. Change blocks setting can change the page layout.

By click on Structure then Blocks, you should see all the block regions available to you. Some block regions is being reserved. For example:

  • Help region - For system help block only
  • Content region - For main page content block only

You can drag & drop or edit the block to publish it to a particular region. Some region can handle more than one blocks, for example left sidebar and right sidebar. Some region can only have one block, for example closure region.

By click on configure link, you can also tell a block when/who to appear. At visibility settings section, you have options of All pages except those listed or Only the listed pages. This is important because it can help you change your page layout. For example, if you set all your left sidebar blocks with All pages except those listed option on a particular page then on that page it will only show right sidebar's blocks. This way you will have a different layout.

4. Have to know - Content types

You have to know about content type because it help you understand node appearance. By click on Structure > Content types, you should see a list of your content types.

Manage fields help you create new field and manage fields order when node being created. If you click on manage fields for Article, you should see four fields Title, Tags, Body and Image. If you click on manage fields for Blog entry, you should only see two fields Title & Body. If you like Blog entry content type to have Tags & Images fields as well as Article content type, you can use Add existing field function. Rearrange the fields will change the fields order when you are creating the node.

Manage display help you control what end user will be seeing. Change order of the fields will change display order to the user. You can also manage your Image field here. For example set difference size of image to default or teaser viewing.

5. Have to know - Image styles

There are three default image styles(size), thumbnail, medium & large. It is being used mostly by content type for teaser and full article display. Find it at Configuration > Media.

6. Have to know - Views

You have to know about Views module because if Drupal is a superhero then Views is his superpower. He hide it under the Structure. That is all you need to know for now. We will explain it in more detail in Omni Feature Setup.

7. Have to know - Site Information

This is where you set your site name, front page number of posts & default path etc. Find it at Configuration > System.

8. Have to know - Performance

Performance is where you set/clear your site Cache. There are also optimization options for your CSS & JavaScript files. Find it at Configuration.
