Omni Basic Setup

- Introduction

In this section, we will show you some basic setup steps that Omni theme need. By using this guide, we assume you have basic knowledge of Drupal and have Drupal 7.x installed on your system.

- Theme Pack Folder Structure

  • omni: theme files folder
    • omni_magazine: this is the actual theme, please upload this folder to your server.
  • licensing
  • modules: all required modules for Omni Drupal theme.
  • document
    • user guide
    • views: importable view files for backup
  • README.txt

- Installation

There is a modules folder within the download package which include all necessary modules for this theme. Only package included version of modules were tested for compatibility. If new version released please backup before upgrade since it might not be compatible.

Please install Omni Magazine theme and following Modules (If you don't know how to install Drupal theme, please check our Drupal Quick Start guide):

  • Block reference 7.x-2.2
  • Chaos tool suite (ctools) 7.x-1.9
  • Entity API 7.x-1.6
  • Entity reference 7.x-1.1
  • Features 7.x-2.7
  • Field Group 7.x-1.4
  • Inline Entity Form 7.x-1.6
  • jQuery Update 7.x-2.7
  • Share Buttons by AddToAny 7.x-4.10
  • Views 7.x-3.13
  • Omni Custom 7.x-1.0
  • Omni Features 7.x-1.0: simply import all content types & views for Omni theme. Available within download files.

- Basic Configuration

You need to do ALL the following steps, even if you think you might not need it for now.

1. Enable required modules BY ORDER!

Go to Modules page:

  1. Enable modules Chaos tools and Bulk Export then click Save Configuration
  2. Enable modules Entity API and Entity tokens then click Save Configuration
  3. Enable modules Block reference, Entity Reference, Field Group and Inline Entity Form then click Save Configuration
  4. Enable modules jQuery Update, Views and Views UI then click Save Configuration
  5. Enable modules Blog, Statistics, AddToAny, Omni Custom and Features then click Save Configuration
  6. Enable modules Omni Features (take longer to install), and Forum then click Save Configuration

2. Configure Statistics

Go back to Modules page and find Statistics module then click on Configure link

  • Check Count content views box then click Save Configuration

3. Configure AddToAny

Go back to Modules page and find AddToAny module then click on Configure link

  • Change Icon size to 16 pixels
  • Adding Service Buttons HTML code
    <a class="a2a_button_facebook"><span class="fa fa-facebook"></span></a>
    <a class="a2a_button_twitter"><span class="fa fa-twitter"></span></a>
    <a class="a2a_button_google_plus"><span class="fa fa-google-plus"></span></a>
    <a class="a2a_button_pinterest"><span class="fa fa-pinterest-p"></span></a>
  • Check Service Buttons placement as Before the universal sharing button
  • Check Universal Button as None
  • Open Placement and check only Article, Blog entry, Display for node teasers, Display in link section, Display in content section
  • Click Save Configuration

4. Configure jQuery Update

Go back to Modules page and find jQuery Update module then click on Configure link

  • Make sure Default jQuery Version as 1.10

5. Check & modify Content Type

Go to Structure > Content types. After all required modules installed, you should see 10 content types:

  1. Article: we need to add an existing field to this content type
    • Click on manage fields link
    • At Add existing field section, find and select field_section from list selection
    • Click Save and continue
    • At setting page, keep it as it is and click Save Setting
    • Back to manage fields, reorder fields as Title > Section > Tags > Images > Body
    • Click Save
    • Click on manage display link
    • Make sure fields show as below for both default & teaser. Image should set to Original image style.

    • Complete
  2. Basic page
  3. Block
    • Click on edit link
    • At Publishing options tab, uncheck Promoted to front page
    • At Display settings tab, uncheck Display author and date information
    • At Comment settings tab, select Closed
  4. Block element
    • Click on edit link
    • At Publishing options tab, uncheck Promoted to front page
    • At Display settings tab, uncheck Display author and date information
    • At Comment settings tab, select Closed
  5. Blog entry: we need to add two existing fields to this content type
    • Click on manage fields link
    • At Add existing field section, find and select field_tags from list selection
    • Click Save and continue
    • At setting page, keep it as it is and click Save Setting
    • Back to manage fields, at Add existing field section, find and select field_image from list selection
    • Click Save and continue
    • At setting page, keep it as it is and click Save Setting
    • Back to manage fields, reorder fields as Title > Tags > Images > Body
    • Click Save
    • Click on manage display link
    • Make sure fields show as below for both default & teaser

    • Complete
  6. Forum topic
  7. Mega menu
    • Click on edit link
    • At Publishing options tab, uncheck Promoted to front page
    • At Display settings tab, uncheck Display author and date information
    • At Comment settings tab, select Closed
  8. Mega menu element
    • Click on edit link
    • At Publishing options tab, uncheck Promoted to front page
    • At Display settings tab, uncheck Display author and date information
    • At Comment settings tab, select Closed
  9. Showcase
    • Click on edit link
    • At Publishing options tab, uncheck Promoted to front page
    • At Display settings tab, uncheck Display author and date information
    • At Comment settings tab, select Closed
  10. Showcase element
    • Click on edit link
    • At Publishing options tab, uncheck Promoted to front page
    • At Display settings tab, uncheck Display author and date information
    • At Comment settings tab, select Closed

6. Set taxonomy terms

Go to Structure > Taxonomy. There should be 3 existing vocabularies:

  1. Forums
  2. Section: we need to adding some sample terms for Section vocabulary
    • Click Add term link and add term News, fill in Description (optional), select a Color then click Save
    • Click Add term link and add term Sport, fill in Description (optional), select a Color then click Save
    • Click Add term link and add term Tech, fill in Description (optional), select a Color then click Save
    • Click Add term link and add term Travel, fill in Description (optional), select a Color then click Save
  3. Tags

7. Check views

Go to Structure > Views. There should be 3 existing Views:

  1. Article Block
  2. Blog Block
  3. Page and Block

We also need to enable a View call Taxonomy term at bottom of the list.

8. Check Image Style

Go to Configuration > Image styles. There should be 6 existing image styles:

  1. Thumbnail (100x100)
  2. Medium (220x220)
  3. Large (480x480)
  4. View large (600x400)
  5. View Medium (300x200)
  6. View Thumbnail (100x80)

9. Setup avatar

Go to Configuration > People > Account setting

  • Check Enable user pictures
  • Fill Picture directory field as pictures
  • Upload avatar.png file in theme file images folder to Picture directory
  • Fill Default picture field with full URL that point to Picture directory
  • Click Save Configuration

10. Publish main menu

Go to Structure > Blocks

  • Go to Structure > Menus to configure Main menu.
  • Go to Structure > Blocks find Main menu block. Publish it at main menu block region.
  • For more detail setup, you can check it out later at Menu Options section

11. Change permission

Go to People > Permissions Tab

  • At Omni Custom section check
    • View full node Blog entry page for ANONYMOUS USER and AUTHENTICATED USER
    • View full node Forum topic page for ANONYMOUS USER and AUTHENTICATED USER
    • View full node Article page for ANONYMOUS USER and AUTHENTICATED USER
    • View full node Basic page for ANONYMOUS USER and AUTHENTICATED USER
  • At Search section check Use search for ANONYMOUS USER
  • Click Save Configuration

12. Omni Magazine theme setting

Go to Appearance

  • Enable and set Omni Magazine as Default theme
  • Make sure administration theme set to Seven
  • Click Setting link under Omni Magazine
  • Color Scheme should set to Default Compact
  • Toggle Display: Check Logo & Slogan OR Site name & Slogan. It can't display Logo & Site Name at same time
  • Omni Setting
    • Header Style: it should match Color Scheme. For example, if you select Color Scheme as Default Box then Header Style should be Box Width Style.
      • Compact Style (default style)
      • Full Width Style
      • Box Width Style
    • Menu Style
      • Drop Down Menu (default style)
      • Mega Menu (see Omni Feature Setup)
      • Hybrid Menu (see Omni Feature Setup)
    • Layout Options: switch sidebars position
      • Primary/Secondary/Tertiary (default style)
      • Secondary/Primary/Tertiary
      • Tertiary/Primary/Secondary
    • Display header search box? (no as default)
    • Display breadcrumb? (no as default)
  • Save the change

Knowing issues of Drupal 7.x

Users might find some functionality involve with Javascript not working properly. To fix it, you might have to enable Aggregate JavaScript files option at Performance. For live site, you should enable Aggregate and compress CSS files too.

Now you have completed the basic setup! You can start to create new content or continue on Omni Feature Setup

- Image Size

With two sidebars layout, default image size for both Article and Blog node should be at least 740px width.
